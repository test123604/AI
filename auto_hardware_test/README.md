# Auto Hardware Test

## 项目简介

这是一个服务器硬件性能自动化测试工具，用于对网吧/游戏服务器（顺网云）进行性能压测和监控。通过远程SSH连接多台服务器，自动部署测试环境、运行压力测试、收集日志并生成Excel测试报告。

## 功能特性

- **远程服务器管理**：支持SSH/SFTP远程连接和管理多台服务器
- **环境自动化部署**：自动上传并解压测试文件，安装基础环境
- **硬件基准测试**：
  - 网卡带宽基准测试（iperf）
  - 磁盘性能基准测试（FIO）
- **压测工具集成**：集成Capture压测工具，模拟真实网吧场景
- **日志自动收集**：自动收集服务器端和客户端日志
- **测试报告生成**：自动解析日志并生成Excel格式的测试报告

## 技术栈

| 技术 | 用途 |
|------|------|
| Python 3 | 主要开发语言 |
| paramiko | SSH/SFTP远程连接 |
| configparser | 配置文件管理 |
| openpyxl | Excel读写操作 |
| multiprocessing | 多进程并发执行 |
| re | 正则表达式日志解析 |

### 核心模块说明

| 模块 | 功能 |
|------|------|
| `Config` | 读取config.conf配置文件，获取服务器连接信息和测试开关 |
| `Multiprocess` | 使用多进程池并发执行远程SSH命令 |
| `Put_Files` | 通过SFTP上传文件到远程服务器或下载远程文件到本地 |
| `Install` | 远程执行环境安装脚本 |
| `BaseDisk` | 生成FIO测试脚本并执行磁盘基准测试 |
| `Network` | 执行iperf网卡带宽测试 |
| `Capture` | 管理Capture压测工具的配置、做包和运行 |
| `Hardware` | 收集服务器硬件信息（CPU、内存、网卡、系统版本） |
| `LogToExcel` | 解析各类日志文件并将结果写入Excel |

## 配置说明

### config.conf 配置文件

```ini
[case]
# 测试服务器类型：read（读服务器）或 write（写服务器）
server=read

# 以下开关设置为1启用，0禁用
upload=1          # 上传并解压压测文件
install=1         # 安装基础环境
test_net=0        # 网卡基准测试
test_disk=0       # 磁盘基准测试
make_package=0    # Capture做包
run_capture=0     # 运行Capture压测

# 磁盘测试路径
data_path=/dev/mapper/mpatha
disk_path=/dev/mapper/mpathb
writeback_path=/dev/mapper/mpathc
test_datadisk_size=57.3

# 服务器配置（ssh1为被测服务器，ssh2/ssh3为模拟客户机）
[ssh1]
bar_num=0
hostname=server45
host=172.17.7.45
port=22
username=root
password=abc#123
timeout=1.0
```

## 使用方法

### 环境依赖

```bash
pip install paramiko openpyxl configparser
```

### 运行测试

```bash
python main.py
```

### 测试建议

1. **首次运行**：设置 `upload=1`, `install=1`，部署测试环境
2. **环境已部署**：设置 `upload=0`, `install=0`，只运行测试
3. **独立运行压测**：完成做包后，可单独设置 `run_capture=1` 运行压测

## 测试报告

测试完成后，将生成 `服务器性能测试结果统计.xlsx`，包含以下工作表：

- **软硬件环境**：服务器IP、CPU、内存、网卡、系统版本
- **网卡基准测试**：带宽测试结果
- **磁盘基准测试**：FIO测试结果（游戏盘/镜像盘/回写盘）
- **capture压测服务端统计**：数据服务、镜像服务、回写服务性能
- **capture压测客户端统计**：各客户端压测结果

## 注意事项

1. 配置文件中的密码请妥善保管，不要提交到版本控制系统
2. 升级内核后需要重启服务器
3. 确保测试服务器之间网络连通
4. 压测前建议清除服务器缓存以获得准确结果
5. 根据实际测试场景调整网吧数量（bar_num）参数
