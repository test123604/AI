# RAG 系统使用说明文档

```

├── rag/
│   ├── __init__.py
```

## 一、代码功能说明

### 1. 核心文件：`rag_system.py`

| 类/方法 | 功能 | 对应 RAG 流程 |
|---------|------|--------------|
| `RAGSystem.__init__()` | 初始化系统，配置嵌入模型和 LLM | 准备阶段 |
| `load_documents()` | 从 data/ 目录加载文档 | 文档加载 |
| `split_documents()` | 将长文档分割成小块 | 文档分割 |
| `create_vectorstore()` | 创建向量数据库 | 向量化 |
| `simple_search()` | 根据问题检索相关文档 | 检索 |
| `generate_answer()` | 基于 LLM 生成回答 | 生成 |
| `query()` | 完整的问答流程 | 端到端 |

### 2. 关键参数说明

```python
# 文档分割参数
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,        # 每块大小（字符数）
    chunk_overlap=50,      # 块之间重叠（保持上下文连续）
    separators=["\n\n", "\n", "。", "!", "?", ".", " ", ""]
)

# 检索参数
retriever = vectorstore.as_retriever(
    search_type="similarity",    # 相似度检索
    search_kwargs={"k": 4}       # 返回最相关的 4 个文档块
)
```

## 二、如何更换知识库资料

### 方法一：替换文档内容

1. 将你的文档放入 `data/` 目录
2. 支持的格式：`.txt`、`.pdf`
3. 删除旧的向量数据库：
   ```bash
   rm -rf chroma_db
   ```
4. 重新运行程序，自动重建索引

### 方法二：添加新文档

直接将新文件放入 `data/` 目录，然后：

```bash
# 删除旧数据库
rm -rf chroma_db

# 重新运行
python simple_demo.py
```

### 文档格式要求

- **TXT 文件**：UTF-8 编码的纯文本
- **PDF 文件**：可扫描的 PDF（支持中文）

## 三、配置使用购买的 LLM

### 1. 编辑 `.env` 文件

```env
# 使用 OpenAI 官方 API
OPENAI_API_KEY=sk-your-openai-key
OPENAI_BASE_URL=https://api.openai.com/v1

# 或使用 Azure OpenAI
OPENAI_API_KEY=your-azure-key
OPENAI_BASE_URL=https://your-resource.openai.azure.com/

# 或使用其他兼容 OpenAI 的服务
OPENAI_API_KEY=your-api-key
OPENAI_BASE_URL=https://your-provider-endpoint
```

### 2. 支持的 LLM 服务商

| 服务商 | BASE_URL 示例 | 说明 |
|--------|--------------|------|
| OpenAI | `https://api.openai.com/v1` | 官方服务 |
| Azure | `https://xxx.openai.azure.com/` | 需要部署端点 |
| 智谱AI | `https://open.bigmodel.cn/api/pas/v1` | 国内推荐 |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | 阿里云 |
| DeepSeek | `https://api.deepseek.com` | 性价比高 |

### 3. 修改使用的模型

编辑 `rag_system.py` 第 55-61 行：

```python
# 初始化 LLM
self.llm = ChatOpenAI(
    model="gpt-4",           # 改成你想要的模型
    temperature=0,
    base_url=os.getenv("OPENAI_BASE_URL"),
    api_key=os.getenv("OPENAI_API_KEY")
)
```

常用模型名称：
- OpenAI: `gpt-4`, `gpt-3.5-turbo`
- 智谱AI: `glm-4`, `glm-3-turbo`
- DeepSeek: `deepseek-chat`, `deepseek-coder`

## 五、常见问题

### Q: 更换文档后需要重新配置吗？
A: 只需删除 `chroma_db` 目录，重新运行即可自动重建。

### Q: 可以用中文文档吗？
A: 完全支持，推荐使用中文文档。

### Q: 如何提高检索准确率？
A:
1. 调整 `chunk_size`（太小丢失上下文，太大不够精确）
2. 增加 `chunk_overlap`（保持信息连续性）
3. 调整检索的 `k` 值（返回更多候选文档）

### Q: API 调用失败怎么办？
A:
1. 检查 `.env` 文件配置
2. 确认 API Key 有效
3. 检查网络连接
4. 确认 BASE_URL 正确

## 六、测试建议

作为 AI 测试工程师，你可以：

1. **准确性测试**：准备 20-50 个测试问题，检查检索和回答质量
2. **性能测试**：记录响应时间、并发处理能力
3. **边界测试**：空文档、超长文档、特殊字符等
4. **幻觉检测**：人工或用 LLM 评判回答是否基于检索内容
